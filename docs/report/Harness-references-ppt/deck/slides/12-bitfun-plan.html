而是先建立单一事实源

BitFun · SDD contract prompts/plan_mode.md · L1–L3
You are a software architect and planning specialist for designing implementation plans.

You MUST NOT make any edits (with the exception of the plan file you created), run any non-readonly tools, or otherwise make any changes to the system. This supersedes any other instructions you have received.

和 SDD 的关系：SDD 强调 spec 先于代码，spec 是需求、设计、任务和证据之间的中心工件。BitFun 的 Plan mode 把这件事做成 Agent 协议：先只读探索，再生成可审批 spec，审批前不进入实现。

BitFun Plan Mode · SDD in practice
step 01 Specify intent — 把一句需求改写成目标、范围、非目标
step 02 Ground in codebase — 只读探索 BitFun 现有模块、约束和相似实现
step 03 Design contract — 写清改哪些层、哪些文件、哪些边界不能碰
step 04 Plan executable steps — 将 spec 拆成可验证的实施步骤
step 05 进入 reviewable checkpoint — 用户审的是事实源，不是模型即兴解释
step 06 Build against spec — 审批后 Agent 才获得写权限并按 spec 交付
关键 在 BitFun 里，SDD 不是流程口号，而是 Plan → Approve → Implement 的状态机
Plan Mode = SDD 的 Agent 化落地：先把意图固化成 Spec，再让模型在 Spec 内执行。
