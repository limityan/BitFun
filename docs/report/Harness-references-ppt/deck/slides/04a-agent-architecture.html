§02.0 · Agent 的基本架构

Agent 只是一个 While 循环

最小循环：思考 → 工具调用 → 观察 → 再思考
直到任务完成或主动停止
Agent control loop
AI Model
只负责生成下一步意图；不直接拥有系统权限。
Agent 框架把模型输出变成可执行、可观察、可回滚的原子步骤。
01 · Think
思考
读取上下文，决定继续推理、调用工具、请求输入或结束。
→
02 · Act
工具调用
通过 tool / MCP / skill 执行动作；前后都有 hook。
→
03 · Observe
观察
工具返回、错误、日志、测试输出进入下一轮上下文。
→
04 · Finish
完成
满足外部证据后结束；否则继续循环或请求人工确认。
Function calling surface
Act 阶段的能力面
Tools
文件、终端、浏览器、测试。
MCP
外部系统统一包装。
Skills
按任务载入已验证流程。
Trace
记录输入、输出和副作用。
原子动作前后都有闸门
Pre-hook
权限检查、危险命令拦截、上下文补全、确认请求。
Post-hook
结果解析、错误回注、状态更新、审计留痕。
Finish 不是一句"完成了"。
它要绑定测试、用户确认、review finding、计划项状态或失败说明这些外部信号。
