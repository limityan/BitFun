Prompt 不只是用户输入的部分

Prompt 定义了每个角色、工具和技能
在当前回合应如何行动
What prompt controls
Prompt 工程是控制模型的起点

在 Agent 系统里，Prompt 分布在多个层级：主 Agent 的任务协议、Sub-Agent 的角色协议、Tool 的调用说明、MCP 的资源语义、Skills 的可复用流程。它们共同决定模型如何解释任务、选择动作、调用能力和停止。

局限也在这里：
Prompt 能约束"模型怎么想和怎么说"，但工具权限、上下文装载、外部验证、审计追踪，必须由更外层 Harness 执行。
Prompt layers inside an agent
主 Agent Promptmain agent
定义整体任务协议：身份、目标、默认工作方式、完成条件、遇到不确定性时是否提问。它决定一轮任务的主控制语气和边界。
Sub-Agent Promptrole agent
把复杂任务拆给专门角色，例如 planner、debugger、reviewer、fixer。每个子角色有自己的输入、输出和禁止事项。
Tool Prompttool schema
告诉模型工具能做什么、参数怎么填、返回值怎么读、失败时怎么处理。好的工具说明会减少误调用和幻觉参数。
MCP Promptexternal context
MCP server 暴露资源和工具，也会携带语义说明：哪些资源可读、哪些操作有副作用、何时需要用户授权。
Skillsstored workflow
Skill 是被沉淀下来的成功流程：何时触发、先读什么、调用什么工具、输出什么格式。它让 Prompt 从单次技巧变成可复用方法。
