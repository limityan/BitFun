§05 / Minimal Harness 13 / 19
§05 · 最小可用 Harness

六步落地清单。

每步对应六维之一
投入按从短到长排列
step01
记忆 ≈ 1 hr
编写 Rules 文件
仓库根目录建 AGENTS.md 或 CLAUDE.md，≤ 50 行覆盖 Setup / Architecture / Style / Don't 四类。模块级冲突时遵循"近者优先"。
step02
编排 ≈ 1 day
区分模式（探索 / 交付）
至少分两种：探索模式（轻 Harness）+ 交付模式（强制先输出 Spec 经审核）。行有余力补 Debug / Review。
step03
反馈 ≈ 1 day
绑定外部验证
在 Rules 中定义验证矩阵（lint / type-check / test / build）。Agent 不可自我宣布完成，失败输出回注上下文进入下一轮。
step04
约束 ≈ 0.5 day
挂危险命令硬拦截 Hook
Claude Code PreToolUse / LangChain wrap_tool_call / Kiro hooks.yaml。最小清单：rm -rf、push --force、DB 写删、production 调用。
step05
评估 ≈ 2 days
建立 Trace 系统
每轮可回放：模型读了哪些文件、输出了哪些 Function Call、工具返回了什么。Agent原子粒度误操作可回滚。
step06
持续 ongoing
成功路径内化为 Skill
将反复出现的成功流程打包为 Skill：该读哪些文件 / 按什么顺序 / 跑什么验证 / 输出什么。Skill = 已验证的 Harness 流程。
六步全部对应六维之一。完成顺序 = 从最短投入开始。
