§03 · 工程划分

Harness 的六个维度。

01
执行Execution
让模型有"手"，能调用外部能力。
实现Function Calling · MCP · Sandbox
02
记忆Memory
让模型记住"该记住的"信息。
实现CLAUDE.md · .cursorrules · 会话持久化 · Skill 沉淀
03
反馈Feedback
让模型知道自己做错了。
实现Lint · Test · CI · Review · 错误回注上下文
04
编排Orchestration
让模型有计划地推进任务。
实现TODO · Spec-Driven Development · 模式切换 · 循环检测
05
约束Constraint
硬拦截危险动作。
实现危险命令清单 · 删除/写库二次确认 · 最小权限 · 用户确认
06
评估Evaluation
调试 Harness 自身。
实现Trace · Eval 集 · 可观测性
下一节四个案例，全部按此六维归因。
注 · 编排与反馈在产品中通常耦合，本框架拆开仅为排错定位
