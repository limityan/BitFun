# 聊天截图与短录屏

BitFun 现在支持在桌面端直接捕获 **单张截图** 或 **短录屏**，并把结果直接附加到当前聊天草稿中。

当前版本支持：
- 桌面端 **截图捕获**
- 最长 **10 秒** 的短录屏
- 录屏会保存为 **一个受管的短视频**
- 发送给多模态模型前，会自动从视频中抽取少量帧进行分析

<!-- page -->

# 如何使用

你可以在聊天输入框中使用新的入口：
- **截图**：立即捕获当前屏幕并附加 1 张图片
- **开始录屏**：启动一次短时录屏
- **停止录屏**：提前结束；如果不手动停止，BitFun 会在 **10 秒** 后自动结束

在附加到消息前，BitFun 会先在本地完成压缩和降采样，这样可以控制存储体积，也能降低多模态模型的额外成本。受管视频会保留在本地会话 artifacts 中用于预览和历史恢复；发送给模型时使用抽取帧和元数据。

# 隐私提醒

截图和录屏里可能包含：
- 密码、验证码
- 通知、私聊消息
- 客户数据或内部业务信息

附加截图或录屏前，BitFun 会先要求隐私确认。只有勾选 **不再提示** 并继续后，才会记住此选择。

你始终保有控制权：
- 不会自动发送
- 不会在后台偷偷录屏
- 不会录制音频
- 发送前可以移除任意截图或录屏
